По поводу работы программы:

0. Автор проекта пишет на маке, поэтому он не может использовать кодировку windows-1251. 
Мне разрешили использовать кодировку UTF-8. Об этом я также прописал и в коде программы.

1. Корректным считается файл: не пустой, без пустых строк, с корректными числами, с корректным именем. 
Если файл не подходит под любой из критериев, пользователь получает сообщение об ошибке с просьбой изменить файл.

2. Реализация программы "в двух словах": пользователь вводит имя, метод ValidateFile() проверяет его на корректность, если он отдает true, то мы с помощью объектов фильтров сортируем все строки в файле. Как это происходит: создаем объект фильтр IntPreciseFilter, если он НЕ возвращает пустую строку (а это происходит когда число дробное), то записываем это число в оба файла. Если пустая строка, то создаем объект-фильтр DoublePreciseFilter, и он уже записывает число только во второй файл (со всеми числами). Пользователю выводится сообщение об успешной фильтрации, предлагается продолжить работу программы.

3. "Тонкости" условия и как я их понял. 
3.1 Если задана точность, например 2, а число 3,144, то число округляется до 2 знаков после запятой, например, 3,14.
3.2 По условию не особо понятно, какие файлы записывать (условие двойственное: в одном месте написано записывайте целые/остальные, в другом целые/все). Я записываю целые/все.
